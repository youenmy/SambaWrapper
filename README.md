# SambaWrapper

Веб-интерфейс для домашней файлопомойки на Ubuntu / Debian — в духе проводника / Synology File Station.
Воткнул USB-диск → открыл `http://<сервер>:8080` → смотришь файлы, расшариваешь папки по Samba, раздаёшь на телевизор по DLNA.

По сути это удобная обёртка над штатными механизмами Linux: `mount` / `ntfs3`, демон `smbd` (Samba) и `minidlna`. Сам сервис их оркестрирует, но в передаче данных не участвует.

## Возможности

- **Двухпанельный файловый менеджер**: дерево дисков и шар слева, файлы справа. Светлая и тёмная тема.
- **Диски**: монтирование/отмонтирование, **автомонтирование по UUID** (диск сам встаёт на старте и при горячей вставке), **форматирование** (ext4 / exFAT / NTFS) с жёстким подтверждением — только для съёмных/USB.
- **Файлы**: создать папку, переименовать, удалить, переместить, загрузка/скачивание через браузер. **Мультивыделение** галочками и **drag-and-drop** перемещение.
- **Samba-шары**: «Расшарить папку» в один клик, права по пользователям (доступ / запись), гость или с паролем.
- **DLNA-медиасервер** (minidlna): включается из UI, телевизор видит сервер «SambaWrapper».
- **Зависшие точки**: если диск выдернули — видно «висящий» маунт, чистится в один клик.
- **Настройки в меню `admin`**: смена пароля, пользователи Samba, смена порта веб-интерфейса.

## Установка

Одной командой (Ubuntu 22.04+ / Debian 12+), интерактивно спросит порт, DLNA и пароль:

```bash
curl -fsSL https://raw.githubusercontent.com/youenmy/SambaWrapper/main/install.sh | sudo bash
```

Или из склонированного репозитория:

```bash
git clone https://github.com/youenmy/SambaWrapper.git
cd SambaWrapper
sudo bash install.sh
```

### Без вопросов (преднастройки через переменные)

```bash
SW_PORT=9000 SW_DLNA=y SW_ADMIN_PASS=secret sudo -E bash install.sh
```

| Переменная | Значение | По умолчанию |
|---|---|---|
| `SW_PORT` | порт веб-интерфейса | `8080` |
| `SW_DLNA` | `y`/`n` — ставить ли DLNA | `y` |
| `SW_ADMIN_PASS` | пароль админа (пусто → сгенерируется) | автогенерация |
| `SW_REPO` | URL репозитория (для curl-бутстрапа) | этот репозиторий |

После установки откроется на `http://<ip>:<порт>`. Логин `admin`, пароль — заданный или из файла `/var/lib/sambawrapper/initial-password.txt`.

## Снос

```bash
sudo bash uninstall.sh           # снять сервис, шары и данные сохранить
sudo bash uninstall.sh --purge   # удалить и шары, и /var/lib/sambawrapper
```

## Где что лежит

| Путь | Что |
|---|---|
| `/opt/sambawrapper/` | код + venv |
| `/var/lib/sambawrapper/` | SQLite, секрет сессий, пароль, `service.env` (порт) |
| `/mnt/sambawrapper/<name>/` | точки монтирования дисков |
| `/etc/samba/smb.conf.d/sambawrapper-*.conf` | управляемые шары |
| `/etc/minidlna.conf` | конфиг DLNA |
| `/etc/sudoers.d/sambawrapper` | разрешённые root-команды сервиса |
| `/etc/systemd/system/sambawrapper.service` | юнит |

## Безопасность

- В админку — пароль (bcrypt), сессии в HttpOnly cookie.
- Сервис работает под непривилегированным пользователем `sambawrapper`; root-операции — через узкий список sudo-команд (mount/umount/format в пределах `/mnt/sambawrapper`, запись `smb.conf.d/sambawrapper-*.conf`, управление smbd/minidlna).
- Форматировать можно только съёмные/USB-диски — системный диск защищён.
- Path traversal в файловом браузере отсечён границей `/mnt/sambawrapper`.

Рассчитано на доверенную домашнюю сеть. На публичный IP светить не нужно.
